# Lalafo.az – Giriş/Qeydiyyat modulunun funksionallıq və qeyri-funksional testi (Test Case / Task 02) 

# Qeyri-Funksional testing - Lalafo.az

## Usability Testing

Lalafo.az saytının qeyri-funksional test edilməsi (Usability Testing) ilkin olaraq Windows 11 mühitində Google chrome vasitəsi ilə aparıldı. Test zamanı saytın giriş və qeydiyyat modulu istifadəçi rahatlığı baxımından yoxlanıldı.

### Giriş bölməsi

1. Saytın giriş/qeydiyyat bölməsi elan yerləşdirmə bölməsinin yanında, ekranın yuxarı sağ küncündə yerləşir.  
Nəticə: Bölmənin yerləşməsi istifadəçi üçün rahatdır və asanlıqla tapılır.  

2. Mouse ilə “Giriş/Qeydiyyat” yazısının üzərinə gəldikdə mətn yaşıl rəngə dəyişir.  
Nəticə: Bu dəyişiklik istifadəçiyə aktiv element üzərində olduğunu göstərir və istifadə rahatlığını artırır.  

3. “Giriş/Qeydiyyat” moduluna klik edildikdə ekranın mərkəzində giriş bölməsi açılır. Bölmənin ölçüləri göz yormayacaq şəkildə və aydın görünəcək formada təqdim olunur.  
Nəticə: bölmənin dizaynı və yerləşməsi istifadəçi üçün rahatdır.  

4. Klik zamanı birbaşa giriş bölməsinin açılması daimi istifadəçilər üçün daha əlverişlidir və əlavə addım tələb etmir.  
Nəticə: Sürətli və rahat giriş imkanı təmin olunur.  

5. Giriş zamanı Facebook, Google, VKontakte və Odnoklassniki kimi fərqli platformalar vasitəsilə daxil olmaq mümkündür. Bu, fərqli platforma istifadəçilərinin rahatlığını nəzərə alır.  
Nəticə: Alternativ giriş üsullarının mövcudluğu istifadəçi təcrübəsini yaxşılaşdırır.  

**Təklif (Improvement):** Mövcud platformalara əlavə olaraq Apple account ilə giriş imkanı da əlavə edilə bilər.  

6. Giriş bölməsində istifadəçinin şəxsi hesabına daxil olması üçün iki əsas xana mövcuddur: login və parol. Login hissəsində nömrə və ya e-poçt seçimi təqdim olunub.  
Nəticə: Bu yanaşma istifadəçi üçün rahat və praktikdir.  

7. Login xanasına klik edildikdə çərçivə yaşıl rəngə dəyişir və alt hissədə kiçik şriftlə düzgün e-poçt ünvanının necə yazılmalı olduğu göstərilir.  
Nəticə: Sistem istifadəçini düzgün formatda məlumat daxil etməyə yönləndirir.  

8. Login xanasına rəqəm daxil edildikdə alt hissədə düzgün telefon nömrəsi formatı nümunə kimi göstərilir. Eyni zamanda istifadəçiyə nömrəni düzgün şəkildə daxil etməyə istiqamət verilir.  
Nəticə: Bu funksiya istifadəçinin səhv məlumat daxil etmə ehtimalını azaldır və istifadədə rahatlıq yaradır.  

9. Parol xanası boş buraxıldıqda qırmızı çərçivə ilə xəbərdarlıq göstərilir, üzərinə klik edildikdə isə çərçivə yaşıl rəngə dəyişir və alt hissədə parolun minimum 5 simvoldan ibarət olması tələbi qeyd olunur.  
Nəticə: Parol tələbləri istifadəçiyə aydın şəkildə təqdim olunur və səhv daxil etmələrin qarşısı alınır.  

10. Xanalar boş buraxıldıqda sistem tərəfindən xəbərdarlıq mesajı göstərilir, xana çərçivəsi qırmızı rəng olur və məlumatın boş buraxılmasının mümkün olmadığı bildirilir.  
Nəticə: İstifadəçi məlumatları düzgün doldurmağa yönləndirilir.  

11.Giriş düyməsi modulun aşağı hissəsində yerləşir, yaşıl rəngdədir və ölçüləri, şrifti baxımından aydın görünür.  
Nəticə: Düymə rahat seçilir və istifadəçi tərəfindən asanlıqla istifadə olunur.  

12. Parolun gizlədilməsi üçün nəzərdə tutulmuş ikon düzgün ölçüdə və uyğun yerdə yerləşdirilmişdir.  
Nəticə: Funksiya rahat şəkildə istifadə olunur.

### Şifrəni unutdum bölməsi

13. “Şifrəni unutdum” yazısı giriş xanasının altında yerləşir  
Nəticə: Yazının şrifti və rəngi aydın görünür və asanlıqla seçilir

Nəticə: Bu yanaşma istifadəçi üçün rahat və praktikdir  

16. Açılan bölmədə input xanası üzərində təlimat yazısı qeyd olunub  
Nəticə: İstifadəçiyə məlumatı düzgün daxil etmə qaydası göstərilir  

17. Login bölməsində olduğu kimi bu xanada da klik zamanı yaşıl və qırmızı rənglərlə vizual dəyişikliklər və düzgün giriş formatı təqdim olunur  
Nəticə: İstifadəçi üçün yönləndirmə aydın olur və səhv daxil etmə ehtimalı azalır

### Qeydiyyat bölməsi (Usability)

18. Yeni istifadəçilər üçün qeydiyyat bölməsinə keçid giriş modulunun yuxarı hissəsində, “Giriş” yazısının yanında yerləşir və asanlıqla seçilə bilir.  
Nəticə: İstifadəçi rahat şəkildə qeydiyyat bölməsinə yönləndirilir.  

19. Yeni istifadəçilər üçün qeydiyyat bölməsi giriş modulunun yuxarı hissəsində, “Giriş” yazısının yanında yerləşir  
Nəticə: İstifadəçi bölməni asanlıqla görür və rahat şəkildə keçid edir

Nəticə: İstifadəçilər üçün sürətli və rahat qeydiyyat imkanı yaradılır  

**Təklif (Improvement):** Apple ilə qeydiyyat imkanı əlavə edilə bilər  

21. Qeydiyyat bölməsində istifadəçi üçün nömrə və parol daxil etmək kifayətdir və iki əsas xana mövcuddur  
Nəticə: Proses sadə və praktikdir  

22. Login və “Şifrəni unutdum” bölməsində olduğu kimi burada da input xanalarında klik zamanı yaşıl, səhv və ya boş buraxıldıqda qırmızı rənglərlə vizual dəyişikliklər və parol tələbləri göstərilir  
Nəticə: İstifadəçi üçün yönləndirmə aydındır və səhvsiz qeydiyyat imkanı yaradır  

23. “Mən robot deyiləm” yoxlama elementi uyğun ölçüdə və düzgün yerdə yerləşdirilib  
Nəticə: Element aydın görünür və istifadəçi tərəfindən rahat istifadə olunur  

24. Qeydiyyat düyməsi aşağı hissədə yerləşir və yaşıl rəngdədir  
Nəticə: Düymənin rəngi, şrifti və ölçüsü göz yormur, aydın görünür və rahat kliklənir

## Compatibility Testing

### Microsoft Edge

1. Microsoft Edge brauzeri ilə lalafo.az saytının qeyri-funksional (compatibility) yoxlanışı aparıldı.  
Yuxarıda 1–24 arası qeyd olunan bütün testlər eyni şəkildə icra edildi.  
Nəticə: Testlərin əksəriyyəti Google Chrome brauzerində əldə olunan nəticələrlə uyğunluq təşkil edir. Lakin 23-cü test zamanı “Mən robot deyiləm” yoxlama elementi mövcud deyil. Bu hal uyğunluq problemi kimi qiymətləndirildi və BUG olaraq qeyd olundu.  

2. Giriş və Qeydiyyat bölməsində parol xanası doldurulduqda əlavə ikinci bir hide/show ikonu yaranır. Bu ikon istifadə edildikdə parol daha böyük şriftlə görünür və ikonun özü də böyük ölçüdə olur.  
Nəticə: Gözlənilməyən UI davranışı müşahidə olunur  
Requirement clarification tələb olunur: İkinci ikonun olması dizayn tələbinə daxildirmi, yoxsa səhvdir — aydınlaşdırılmalıdır.

Gözlənilən nəticə: xanalar qırmızı rənglə vurğulanmalı, şərtlər göstərilməli və qeydiyyat mümkün olmamalıdır  
Nəticə: xanalar qırmızı rənglə vurğulandı, şərtlər göstərildi və qeydiyyat mümkün olmadı  

2. Nömrə və parol xanaları doldurulub, lakin müqavilə şərtləri ikonu işarələnmədən qeydiyyat cəhdi  
Gözlənilən nəticə: qeydiyyat mümkün olmamalı və müqavilə şərtlərinin seçilməsi tələb olunmalıdır  
Nəticə: qeydiyyat mümkün olmadı və müqavilə şərtlərinin seçilməsi ilə bağlı xəbərdarlıq göstərildi  

3. Düzgün nömrə və səhv formatda parol (5-dən az simvol) ilə qeydiyyat cəhdi  
Gözlənilən nəticə: xana qırmızı rənglə vurğulanmalı və qeydiyyat mümkün olmamalıdır  
Nəticə: qırmızı rəng və xəbərdarlıq göstərildi, qeydiyyat mümkün olmadı  

4. Nömrə xanası boş, parol xanası dolu halda qeydiyyat cəhdi  
Gözlənilən nəticə: nömrə xanası qırmızı rənglə vurğulanmalı və qeydiyyat mümkün olmamalıdır  
Nəticə: qeydiyyat mümkün olmadı və nömrə xanası üçün xəbərdarlıq göstərildi  

5. Düzgün nömrə və parol daxil edilib, lakin müqavilə şərtləri ikonu işarələnməyib  
Gözlənilən nəticə: qeydiyyat mümkün olmamalı və bu şərtin doldurulması tələb olunmalıdır  
Nəticə: qeydiyyat mümkün olmadı və uyğun xəbərdarlıq göstərildi  

6. Nömrə xanasına həddindən artıq (sonsuz) simvol daxil edilməsi (qeydiyyat butonu klik edilmədən)  
Gözlənilən nəticə: nömrə üçün simvol limiti olmalı və artıq simvollar qəbul edilməməlidir  
Nəticə: sistem limit tətbiq etdi və artıq simvollar qəbul edilmədi
